Introduction to Information Security
January 4, 2016

Speaking of information security, the general concern is about virus and malware. These two are a subset of all the threats. Before we explore them all, take a look at this link published by the website www.informationisbeautiful.net. It's absolutely astonishing to almost anyone who didn't see it before. In this blog, we are going to dive into information security and talk about how they are manifested at four different layers: network, system, application and data.

More...

In the enterprise world, information security involves network, software, hardware, data, operating personnel and operation procedures. the challenges and solutions can be broken into the following categories:

  • Network Security: viruses and malware traverse through the network and attack operating system, software, hardware and data storage. It also causes harm to the normal network operation. The solution typically relies on network security appliances inspecting the traffic pattern and compare that with previously identified virus/malware signatures. Once positively identified, the appliance will block the malicious content from getting into the next layer system. The appliances typically are firewall, anti-virus, anti-malware and intrusion prevention systems. Traditional solutions are far from satisfactory as up to 90% alert are false alert, according to a recent Gartner report.

  • System and Data Security: virus and malware can cause system and data corruption, resulting in service and data loss. Another source of threat comes from external hacker and corporate insiders who steal or change sensitive information. To prevent these threat, company should keep operating system and anti-virus software up to date, deploy disaster recover site, encrypt sensitive data and properly manage data backup.

  • Information Security Management: provides account, authentication, authorization and auditing services. The subjects under this category include end user, service account and API. Account service is very business specific and responsible for the subjects' life cycle management. Sometimes it's also called user provisioning or user management. Authentication has many different methods, including but not limited to, username/password, kerberos, x509 certificate, ip address, biometric methods (eye print, iris, fingerprint and voice). Authentication can be further enhanced with a second factor, such as one time toke. Auditing is used to record what people did where and at what time. It's an important aspect for compliance management. API Gateway centralizes API management. A business can build an API once and used by different clients in a standard deployment. The growth in this area has been driven by mobile application development. API gateway also provides a single point for policy enforcement, auditing, logging and other functionalities. It greatly simplifies the development and deployment.

  • Policy Management: relates to the common and business specific security policy and procedures. It offers the capability to dynamically enforce the policies in runtime workflow. All three areas above overlap in this area.
